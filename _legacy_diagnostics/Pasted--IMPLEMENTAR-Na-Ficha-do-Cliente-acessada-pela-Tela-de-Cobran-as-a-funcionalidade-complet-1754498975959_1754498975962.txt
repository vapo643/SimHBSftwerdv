**IMPLEMENTAR:** Na "Ficha do Cliente" (acessada pela "Tela de Cobranças"), a funcionalidade completa de "Histórico de Observações", permitindo à equipe de cobranças registrar e visualizar anotações sobre as propostas.

**CONTEXT:** A equipe precisa de um sistema de registro para acompanhar as interações e negociações com os clientes. Este histórico é fundamental para a organização interna e para a auditoria dos processos de cobrança.

**CURRENT STATE:** A Ficha do Cliente não possui uma seção para adicionar ou listar observações.

**EXPECTED (Estado Final de Sucesso):**

**1. Componentes da Interface (UI):**
    - Na Ficha do Cliente, abaixo dos detalhes principais, crie uma nova seção chamada "Histórico de Observações".
    - Esta seção deve conter:
        - Um campo de texto (`textarea`) grande para o usuário digitar a observação.
        - Um menu dropdown (`select`) para definir um "Status" para a observação. As opções devem ser: `Contato Realizado`, `Negociação em Andamento`, `Acordo Fechado`, `Monitoramento`, `Outros`.
        - Um botão "Salvar Observação".
    - Abaixo dos campos de entrada, deve ser exibida uma lista com todas as observações já salvas para aquela proposta, ordenada da mais recente para a mais antiga.
    - Cada item na lista do histórico deve exibir claramente: a **mensagem**, o **status**, o **email do usuário** que a criou e a **data/hora** do registro.

**2. Lógica de Funcionamento:**
    - Ao clicar em "Salvar Observação", o texto e o status selecionado devem ser enviados para o backend.
    - Após salvar com sucesso no banco de dados, a nova observação deve aparecer instantaneamente no topo da lista do histórico na UI, e os campos de texto e status devem ser limpos, prontos para um novo registro.
    - Ao carregar a Ficha do Cliente, o sistema deve buscar e exibir todo o histórico de observações existente para aquela proposta.

**INTEGRAÇÃO:**
- Crie dois novos endpoints no backend:
    1.  `GET /api/propostas/:propostaId/observacoes` para buscar o histórico.
    2.  `POST /api/propostas/:propostaId/observacoes` para salvar uma nova observação.
- O backend deve popular o campo `criado_por` automaticamente com o email do usuário autenticado.

**ROLES:** Usuários com as roles `ADMIN` e `COBRANÇA` devem ter permissão total para adicionar e visualizar as observações.

**BANCO (Diretriz Crítica):**
- Utilize a tabela `historico_observacoes_cobranca` que já existe no nosso banco de dados.
- Garanta que sua implementação insira e leia os dados corretamente desta tabela, associando cada observação ao `proposta_id` correto.

**TESTE:**
1.  Abra a Ficha de um Cliente e confirme que a nova seção "Histórico de Observações" é exibida.
2.  Escreva uma observação, selecione um status e clique em "Salvar".
3.  Valide se a observação foi salva corretamente na tabela `historico_observacoes_cobranca` no banco de dados.
4.  Confirme que a nova observação aparece no topo da lista na UI.
5.  Recarregue a página e verifique se o histórico completo daquela proposta é carregado e exibido corretamente.
