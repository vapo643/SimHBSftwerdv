## Entendendo o Comportamento Anômalo da API

O que você está observando é realmente **muito incomum** para uma API REST bem implementada. Quando uma API retorna HTTP 200 OK com `content-type: application/json` em um endpoint que deveria retornar PDF, isso geralmente indica uma de três situações:

**Primeira possibilidade**: A API implementou um sistema de **duas etapas** onde o primeiro endpoint apenas inicia o processo de geração do PDF, retornando informações sobre onde buscá-lo depois.

**Segunda possibilidade**: Existe uma **configuração específica da conta** ou um **header proprietário** não documentado que controla o formato da resposta.

**Terceira possibilidade**: A API mudou recentemente seu comportamento, mas manteve compatibilidade retornando dados em JSON que podem ser processados de forma diferente.
