**### Pacote de Ativação de Missão (PAM) V1.0**

* **Sumário da Missão (O Quê):** Realizar uma auditoria forense do fluxo de geração de boletos para mapear o ponto exato onde os dados do PIX (`pixCopiaECola`) são recebidos e onde eles deveriam ser persistidos em nosso banco de dados.
* **Intenção Estratégica (O Porquê):** O Arquiteto definiu uma nova arquitetura onde os dados do PIX devem ser salvos em nosso banco no momento da geração. Antes de implementar, precisamos entender o fluxo de dados atual para planejar a modificação cirúrgica.
* **Histórico Relevante (Consulta Obrigatória):** O problema atual é a exibição inconsistente do botão "Copiar PIX". A causa raiz é a dependência de chamadas em tempo real. A nova arquitetura resolve isso ao persistir os dados.
* **Modelo Mental (Como se Encaixa):** Você é um auditor de dados. Sua tarefa é seguir o rastro do `pixCopiaECola` desde a sua origem (a resposta da API do Banco Inter) até o seu destino final (a UI), e identificar o ponto onde a corrente de dados é quebrada (ou seja, onde a persistência no banco de dados falha em ocorrer).
* **Riscos Antecipados:** **Risco:** Uma análise superficial. **Contramedida:** Sua missão não é resolver, mas sim **PROVAR**, com evidências de código, onde a persistência precisa ser adicionada.

**3. Auditoria do Schema do Banco de Dados:**

**Relatório Final:**
O critério de sucesso é um relatório que nos dê um mapa claro de todas as modificações necessárias para implementar a nova arquitetura de persistência de dados do PIX.
