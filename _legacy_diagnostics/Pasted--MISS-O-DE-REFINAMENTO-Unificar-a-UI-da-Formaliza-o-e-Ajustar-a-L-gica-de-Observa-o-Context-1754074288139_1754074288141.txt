# MISSÃO DE REFINAMENTO: Unificar a UI da Formalização e Ajustar a Lógica de Observação

**Contexto:** Olá, Agente. O nosso sistema está a funcionar, mas encontrámos duas inconsistências importantes na interface e na lógica de negócio que precisam de ser corrigidas para que a experiência do usuário seja perfeita e alinhada com as nossas regras.

**AÇÃO:**
Execute o seguinte roadmap de 2 fases.

**ROADMAP DE EXECUÇÃO:**

**FASE 1: Unifique a Visualização da Tela de Formalização**
* **O Problema (Sintoma):** Atualmente, a tela de "Formalização" parece diferente para um `ATENDENTE` e para um `ADMINISTRADOR`. A visão do atendente está limitada e não tem todas as funcionalidades.
* **A Regra de Negócio:** A tela deve ser **idêntica** para ambos os perfis. A única diferença é nos *dados* que eles veem (o Admin vê tudo, o Atendente só vê o dele, conforme o nosso RLS), mas os componentes, botões e funcionalidades na tela devem ser os mesmos.
* **Sua Tarefa (Frontend):** Refatore o nosso componente `/client/src/pages/formalizacao.tsx`. Remova qualquer lógica de renderização condicional que esteja a mostrar uma UI diferente ou limitada para o `ATENDENTE`. Garanta que ambos os perfis vejam exatamente a mesma interface.

**FASE 2: Corrija a Obrigatoriedade da Observação do Analista**
* **O Problema (Sintoma):** Na tela de "Análise de Proposta" (T-03), quando o `ANALISTA` vai tomar uma decisão, o campo "Observação" está a ser exigido como obrigatório para todas as ações (Aprovar, Negar e Pendenciar).
* **A Regra de Negócio:** A regra correta é: a observação é **obrigatória APENAS** quando o status é mudado para **`pendente`**. Para os status `aprovado` e `negado`, o campo de observação é **opcional**.
* **Sua Tarefa (Frontend):** Modifique a lógica de validação do formulário no modal de decisão do analista. A validação (provavelmente com Zod) deve ser dinâmica: aplique a regra de "obrigatório" ao campo `observacao` somente se a ação selecionada for "Pendenciar".

Execute este plano para refinar e alinhar a nossa aplicação com as regras de negócio.
