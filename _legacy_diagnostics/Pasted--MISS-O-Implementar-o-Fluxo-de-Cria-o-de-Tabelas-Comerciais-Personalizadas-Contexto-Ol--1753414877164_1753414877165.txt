# MISSÃO: Implementar o Fluxo de Criação de Tabelas Comerciais Personalizadas

**Contexto:** Olá, Agente. O nosso sistema está estável, e agora precisamos de implementar uma das lógicas de negócio mais importantes: a capacidade de um Administrador criar uma Tabela Comercial personalizada para um parceiro específico.

**A Sua Missão (O Fluxo do Usuário):**

A sua tarefa é dar vida ao fluxo de personalização na tela de "Detalhes do Parceiro". O fluxo que você precisa construir é o seguinte:

1.  **O Ponto de Partida:** O Administrador está na página de **"Detalhes do Parceiro"** (ex: a página do "Parceiro A").

2.  **A Ação:** Dentro da seção "Configuração Comercial", ele interage com o dropdown de "Tabela Comercial Principal".

3.  **A Nova Funcionalidade:** Nós precisamos que você adicione uma opção especial e estática no topo deste dropdown, com o texto: **"-- Criar Tabela Personalizada --"**.

4.  **A "Magia" (O Modal):** Quando o Administrador seleciona esta nova opção, um **modal** deve ser aberto.

5.  **A Lógica do Modal (O Ponto Crítico):**
    * O modal que se abre deve ser o mesmo formulário que usamos para criar uma Tabela Comercial na tela de "Gestão de Tabelas Comerciais" (Tela 3).
    * **Detalhe Essencial:** Como esta tabela é *personalizada*, o campo que associa a tabela a um Parceiro (`parceiro_id`) dentro deste formulário deve vir **automaticamente preenchido com o ID do parceiro** cuja página de detalhes estamos a ver (no nosso exemplo, o ID do "Parceiro A"). Este campo deve estar **bloqueado (read-only)**, para que o Administrador não o possa alterar.

**Resumo da Missão:**
Implemente esta funcionalidade. Ao final, um Administrador deve ser capaz de ir à página de detalhes de um parceiro, selecionar a opção de criar uma tabela personalizada, preencher o formulário (onde o parceiro já está definido e bloqueado) e, ao salvar, criar uma nova Tabela Comercial no banco de dados que está corretamente atrelada tanto a um `Produto` quanto àquele `Parceiro` específico.
