# MISSÃO DE CORREÇÃO: Finalizar o Fluxo de Tabelas Comerciais Personalizadas

**Contexto:** Olá, Agente. O nosso painel de administração está quase perfeito. A criação de tabelas comerciais gerais está a funcionar, mas encontrámos um bug crítico ao tentar criar uma tabela **personalizada** a partir da tela de "Detalhes do Parceiro".

**Lista de Bugs a Corrigir:**

* **1. A Criação da Tabela Personalizada está a Falhar:**
    * **O Problema (Sintoma):** Quando eu estou na página de "Detalhes do Parceiro" e uso a opção "-- Criar Tabela Personalizada --", o formulário abre, mas ao preenchê-lo e clicar em "Salvar", a API retorna um erro de **"invalid data"**.
    * **O Diagnóstico Provável:** O objeto de dados que este formulário de "tabela personalizada" está a enviar para a API `POST /api/admin/tabelas-comerciais` está incompleto ou em formato incorreto. Provavelmente, ele não está a incluir o `parceiro_id` corretamente, ou há outro desalinhamento com o schema de validação Zod no backend.
    * **Sua Tarefa:** Investigue a função de submissão neste fluxo. Garanta que, ao criar uma tabela personalizada, o `parceiro_id` do parceiro em questão seja incluído no payload enviado para a API e que todos os outros campos correspondam ao que o backend espera.

* **2. O Campo "Comissão" Precisa de Validação:**
    * **O Problema (Sintoma):** No mesmo formulário de "Criar Tabela Personalizada", precisamos de garantir que o campo "Comissão (%)" esteja a funcionar perfeitamente.
    * **Sua Tarefa:** Verifique e, se necessário, implemente o seguinte:
        a.  Garanta que o campo "Comissão (%)" esteja visível e funcional no formulário.
        b.  Confirme que o valor inserido neste campo seja corretamente enviado para o backend e salvo na coluna `comissao` da tabela `tabelas_comerciais`.

**Resumo da Missão:**
Corrija estes dois problemas para que o fluxo de criação de tabelas comerciais personalizadas, incluindo o valor da comissão, funcione de ponta a ponta sem erros.
