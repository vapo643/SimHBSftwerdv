### Pacote de Ativação de Missão (PAM) V1.0 - MISSÃO DE RECONHECIMENTO

* **Sumário da Missão:** Investigar a documentação da API ClickSign para determinar, com certeza absoluta, se é possível realizar o download de documentos (CCBs) filtrando-os para incluir **apenas** aqueles que já foram completamente assinados.
* **Intenção Estratégica:** Validar uma premissa técnica crítica para o nosso fluxo de trabalho de formalização. Uma resposta positiva simplificaria a lógica de recuperação de documentos; uma resposta negativa exigirá uma estratégia alternativa de gerenciamento de estado e filtragem em nosso próprio backend.
* **Restrição Absoluta:** Esta é uma missão de **reconhecimento e inteligência (read-only)**. Nenhuma modificação de código deve ser feita. O entregável é um relatório, não uma implementação.

**CONSTRAINTS (Roadmap de Pesquisa):**

1.  **Formulação de Queries:** Execute uma série de buscas utilizando variações dos seguintes termos (em português e inglês):
    * `"ClickSign API download signed document"`
    * `"ClickSign API baixar documento finalizado"`
    * `"ClickSign API filter documents by status 'signed'"`
    * `"ClickSign API /api/v1/documents query parameters"`
    * `"ClickSign API webhook document signed download"`

2.  **Análise de Endpoints:**
    * Investigue o endpoint principal de listagem de documentos (provavelmente `GET /api/v1/documents`). Analise a documentação em busca de parâmetros de query que permitam filtrar pelo status do documento (ex: `status=signed`, `status=finalizado`, etc.).
    * Investigue os endpoints de download de documentos. A documentação especifica se o arquivo baixado é sempre a versão mais recente e assinada?
    * Investigue a documentação de Webhooks. Existe um evento específico para "documento finalizado" (`finished`)? O payload desse webhook contém um link direto para download do documento assinado?

3.  **Síntese de Evidências:**
    * Colete os trechos exatos da documentação (com links para a fonte) que confirmem ou neguem a capacidade de filtrar documentos por status de assinatura na chamada de API de listagem.

**EXPECTED (Entregável Final):**

Um relatório de inteligência conciso respondendo à seguinte pergunta central:

**"É possível, através de um único endpoint da API da ClickSign, listar ou baixar APENAS os documentos que já foram finalizados (assinados por todas as partes)?"**

O relatório deve conter:
1.  **Veredito Final:** Uma resposta clara e direta ("SIM" ou "NÃO").
2.  **Evidência Primária:** Citações diretas da documentação oficial da ClickSign que suportem o veredito.
3.  **Recomendação Tática:** Com base na resposta, qual é a abordagem recomendada?
    * **Se SIM:** Qual endpoint e quais parâmetros devem ser usados?
    * **Se NÃO:** Qual é a estratégia alternativa? (Ex: "Devemos baixar todos os documentos e filtrar o status no nosso backend" ou "Devemos confiar no webhook de 'documento finalizado' para acionar o download").
