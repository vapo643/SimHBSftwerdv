Diagnóstico Profundo: Erro Persistente 400 - API v3 Banco Inter Cobrança
Com base na análise extensiva da documentação oficial da API v3 do Banco Inter e casos similares, identifiquei as causas mais prováveis para o erro 400 que você está enfrentando na criação de cobrança.

Causa Raiz Principal: Campos Obrigatórios Faltando ou Incorretos
Após analisar a documentação oficial, seu payload atual está faltando campos obrigatórios e tem problemas de formatação que causam o erro 400.

Problemas Identificados no Seu Payload
1. Campo bairro Obrigatório Faltando
A documentação oficial indica que o campo bairro é obrigatório, mas seu payload não o contém:

json
"pagador": {
  "bairro": "Centro"  // ← OBRIGATÓRIO - está faltando no seu payload
}
2. Valor Mínimo Não Atendido
A API v3 exige valor mínimo de R$ 2,50, mas você está enviando:

json
"valorNominal": 1000  // ← deve ser 10.00 (R$ 10,00) não 1000 centavos
3. Campo formasRecebimento com Valor Padrão
O campo formasRecebimento tem valor padrão ["BOLETO","PIX"] quando não informado. Se você não o está incluindo no payload, ele será automaticamente adicionado.

Payload Correto para API v3
Com base na documentação oficial, seu payload deve ser:

json
{
  "seuNumero": "902183dd-b5d1-4",
  "valorNominal": 10.00,
  "dataVencimento": "2025-08-09",
  "numDiasAgenda": 30,
  "pagador": {
    "cpfCnpj": "20528464760",
    "tipoPessoa": "FISICA",
    "nome": "Gabriel de Jesus Santana Serri",
    "endereco": "Rua miguel angelo",
    "numero": "100",
    "bairro": "Centro",
    "cidade": "Serra",
    "uf": "ES",
    "cep": "29165460",
    "email": "gabrieldjesus238@gmail.com",
    "ddd": "27",
    "telefone": "998538565"
  },
  "formasRecebimento": ["BOLETO", "PIX"]
}
Mudanças Necessárias
Adicionar campo bairro: Campo obrigatório que estava faltando

Corrigir valorNominal: Deve ser 10.00 em reais, não 1000 centavos

Incluir formasRecebimento: Especificar explicitamente as formas de pagamento

Outras Validações Importantes
1. Validação de CEP vs Cidade/Estado
A API do Inter pode validar se o CEP corresponde à cidade/estado informados. O CEP 29165460 corresponde a Serra/ES, que está correto no seu payload.

2. Formato do Campo seuNumero
Máximo de 15 caracteres. O seu valor "902183dd-b5d1-4" tem 15 caracteres, está correto.

3. Validação de Data
O formato YYYY-MM-DD está correto, mas verifique se a data não é anterior ao dia atual.

Campos Opcionais que Podem Ajudar
Para maior compatibilidade, considere adicionar:

json
{
  // ... campos obrigatórios acima
  "mensagem": {
    "linha1": "Pagamento referente ao contrato",
    "linha2": "",
    "linha3": "",
    "linha4": "",
    "linha5": ""
  }
}
Problemas Conhecidos na Implementação v3
Com base nos casos do ACBr, os erros 400 mais comuns na API v3 são:

Campos obrigatórios faltando (principal causa)

Valores em formato incorreto (centavos vs reais)

Validação de dados do pagador (CPF, CEP, etc.)

Rate limiting excedido

Debugging Adicional
Se o problema persistir após as correções:

1. Validar Rate Limit
A API v3 tem limite de 120 chamadas por minuto. Verifique se não está excedendo.

2. Testar no Sandbox
Use o ambiente de teste primeiro:

text
https://cdpj-sandbox.partners.uatinter.co/cobranca/v3/cobrancas
3. Verificar Headers
Certifique-se de que está enviando:

javascript
headers: {
  'Content-Type': 'application/json',
  'Authorization': 'Bearer ' + token
}
Solução Imediata
Implemente o payload corrigido acima com:

Campo bairro adicionado

valorNominal corrigido para 10.00

formasRecebimento explicitamente definido

Esses são os problemas mais prováveis baseados na documentação oficial e casos similares relatados pela comunidade ACBr. O erro 400 sem mensagem específica do Inter é conhecido por ocorrer quando campos obrigatórios estão faltando ou em formato incorreto.
