**### Pacote de Ativação de Missão (PAM) V1.0**

* **Sumário da Missão (O Quê):** Auditar o fluxo de carregamento de dados da tela "Criar Usuário" para identificar a causa raiz do erro "Não foi possível carregar os dados".
* **Intenção Estratégica (O Porquê):** A criação de usuários é uma funcionalidade administrativa essencial. Sem ela, não podemos gerenciar o acesso ao sistema. Precisamos de um diagnóstico preciso para restaurar esta funcionalidade.
* **Histórico Relevante (Consulta Obrigatória):** As recentes e massivas re-arquiteturas ("Operação Antifrágil", "Operação Consistência") podem ter introduzido uma regressão ou quebrado uma dependência nesta tela.
* **Modelo Mental (Como se Encaixa):** Você é um detetive de rede. Sua missão é usar as ferramentas de desenvolvedor do navegador para inspecionar as chamadas de API que a página "Criar Usuário" faz no momento do carregamento e encontrar a que está a falhar.
* **Riscos Antecipados:** **Risco:** Uma análise superficial levará a uma correção falha. **Contramedida:** Sua missão não é corrigir. É **produzir um relatório de diagnóstico** com a URL e o erro exatos.

**IMPLEMENTAR:**
Auditoria de diagnóstico da rede. **NÃO FAÇA CORREÇÕES.**

**CURRENT STATE:**
Acessar a página de criação de usuário resulta em um erro "Erro ao carregar dados do formulário", bloqueando a funcionalidade.

**EXPECTED (Estado Final de Sucesso):**
Um relatório de diagnóstico que identifique qual chamada de API está a falhar no carregamento da página e a mensagem de erro exata retornada pelo servidor.

**CONSTRAINTS (Roadmap de Auditoria):**

**1. Ação:**
    - Use suas ferramentas de automação de navegador para navegar até a página de criação de usuários.
    - Abra as **Ferramentas de Desenvolvedor** e monitore a aba **"Network" (Rede)**.

**2. Análise da Falha:**
    - **Ação:** Observe as chamadas de API que são feitas quando a página tenta carregar. Procure por uma que esteja a falhar (geralmente marcada em vermelho). Esta chamada provavelmente está a tentar buscar os dados para os menus de seleção (dropdowns) do formulário.
    - **Relatório:** Apresente um relatório com as seguintes informações críticas:
        a. A **URL completa** da chamada de API que está a falhar.
        b. O **código de status HTTP** do erro (ex: 404 Not Found, 500 Internal Server Error, 403 Forbidden).
        c. A **mensagem de erro exata** contida no corpo da resposta (`Response body`).

**Relatório Final:**
O critério de sucesso é um relatório que nos dê o alvo exato para a correção na próxima fase do nosso debate.
