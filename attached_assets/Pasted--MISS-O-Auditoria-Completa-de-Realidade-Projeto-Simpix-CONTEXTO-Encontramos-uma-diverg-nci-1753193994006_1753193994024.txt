# MISSÃO: Auditoria Completa de Realidade - Projeto Simpix

**CONTEXTO:** Encontramos uma divergência crítica entre o progresso reportado pelo Agente e o estado real da aplicação. Suspeitamos que o Agente pode ter criado e utilizado um banco de dados local do Replit em vez da nossa instância oficial do Supabase. Precisamos de uma auditoria completa para estabelecer a "verdade de campo".

**AÇÃO:** Sua tarefa é realizar uma auditoria de 3 camadas e nos fornecer um relatório detalhado.

---
**CAMADA 1: AUDITORIA DE CONEXÃO DE DADOS**

1.  **Verifique os "Secrets":** Analise as variáveis de ambiente. As credenciais (`DATABASE_URL`, `SUPABASE_URL`) apontam para a nossa instância do Supabase ou para um banco de dados local do Replit?
2.  **Verifique o Código de Conexão:** Analise os arquivos (`server/lib/supabase.ts`, `server/storage.ts`) que estabelecem a conexão com o banco. Eles estão configurados para usar as credenciais do Supabase dos "Secrets"?

**RELATÓRIO - CAMADA 1:** O software está, de fato, conectado ao banco de dados Supabase em todas as suas partes? (SIM/NÃO)

---
**CAMADA 2: AUDITORIA DE SCHEMA DE BANCO DE DADOS**

1.  **Execute uma Consulta:** Conecte-se ao banco de dados que a aplicação está usando e liste todas as tabelas existentes no schema 'public'.
2.  **Compare com o Blueprint:** Compare a lista de tabelas que você encontrou com a lista de tabelas que deveriam existir, conforme nosso `Blueprint v4.3` (`parceiros`, `lojas`, `produtos`, `tabelas_comerciais`, `profiles`, `propostas`, `proposta_logs`, `parcelas`).

**RELATÓRIO - CAMADA 2:** Todas as 8 tabelas do nosso `Blueprint` existem no banco de dados que a aplicação está utilizando? (SIM/NÃO)
