### **FASE 1: ALINHAMENTO ESTRATÉGICO (QUESTIONÁRIO MANDATÓRIO - QAE)**

**Objetivo:** Extrair 100% das regras de negócio, restrições e intenção estratégica para a "Missão Simpix".

Por favor, responda detalhadamente aos 5 eixos abaixo:

#### **Eixo I: Negócio (O "Valor")**
1.  Qual é o problema de negócio principal que a "Missão Simpix" se propõe a resolver para o usuário final?
2.  Qual é a métrica de sucesso primária para esta missão? (Ex: Aumento de X% na conversão, redução de Y% no tempo de suporte, etc.)
3.  Existem regras de negócio "inquebráveis", exceções ou lógicas financeiras/críticas que são o núcleo desta missão? Descreva-as.

#### **Eixo II: Usuário (A "Experiência")**
1.  Quais são os papéis de usuário (Roles) que irão interagir com a funcionalidade resultante da "Missão Simpix"?
2.  Descreva o "caminho feliz" (happy path) ideal do usuário, passo a passo, do início ao fim da interação.
3.  Qual é o "caminho infeliz" (unhappy path) mais provável ou mais crítico que devemos prever e tratar? (Ex: Falha de pagamento, dados inválidos, etc.)

#### **Eixo III: Técnico (O "Como")**
1.  A "Missão Simpix" possui dependências mandatórias de tecnologias, serviços externos ou APIs específicas? (Ex: Requer integração com a API X, deve usar o banco de dados Y, etc.)
2.  Como esta nova funcionalidade deve interagir com os módulos ou sistemas já existentes? Existem contratos de API ou fluxos de dados que precisam ser respeitados?

#### **Eixo IV: Risco (As "Armadilhas")**
1.  Na sua avaliação, qual é o maior risco (técnico, de negócio ou de experiência do usuário) para esta implementação?
2.  Qual é a "Definição de Feito" (Definition of Done) inequívoca para esta missão? O que precisa ser entregue para que você a considere concluída?

#### **Eixo V: Sustentabilidade e Operações (A "Vida Longa")**
1.  Como saberemos proativamente que esta funcionalidade está a funcionar corretamente em produção? Quais logs essenciais, métricas ou KPIs de monitoramento devemos implementar?
2.  Qual é a estratégia de testes que garantirá a qualidade a longo prazo (unitários, integração, E2E)?
3.  Esta funcionalidade é crítica a ponto de necessitar de uma "feature flag" para desativação instantânea em caso de falha em produção?
