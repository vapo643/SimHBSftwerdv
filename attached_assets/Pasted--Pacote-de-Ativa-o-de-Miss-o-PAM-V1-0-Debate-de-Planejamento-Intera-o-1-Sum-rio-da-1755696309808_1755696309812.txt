* **Histórico Relevante (Consulta Obrigatória):** A próxima grande missão é garantir a consistência dos dados em todo o fluxo da aplicação, começando pela tela de "Nova Proposta". O Arquiteto Chefe definiu uma abordagem "Frontend-first".

**TASK (Sua Tarefa):**
Abaixo está o roadmap proposto para a "Missão: Auditoria de Consistência de Dados V2". Analise-o criticamente e responda às perguntas que se seguem.

**Roadmap Proposto:**
* **FASE 1: Auditoria do Ponto de Captura (Frontend):** Mapear todos os campos de dados preenchidos pelo usuário no formulário de "Nova Proposta".
* **FASE 2: Auditoria da Camada de Persistência (Backend -> DB):** Rastrear cada campo do frontend através da API até a sua coluna no banco de dados.
* **FASE 3: Auditoria do Ciclo de Vida dos Dados (Fluxos de Negócio):** Verificar a consistência dos dados principais através dos estágios de negócio (análise, CCB, pagamento, etc.).

**Perguntas para Debate (Responda a cada uma):**

1.  **Análise Estratégica:** A abordagem "Frontend-first" descrita neste roadmap é, na sua opinião, a estratégia mais eficaz e segura para auditar a consistência de dados neste sistema? Porquê?
2.  **Análise de Risco:** Quais são os 2 maiores riscos ou "pontos cegos" que você antecipa com esta estratégia? (Ex: "Risco de que a lógica de transformação de dados no backend seja complexa e difícil de rastrear", "Ponto cego: os jobs assíncronos podem modificar os dados de formas não óbvias").
3.  **Sugestão de Refinamento:** Existe alguma fase ou passo que você adicionaria, removeria ou modificaria neste roadmap para torná-lo mais robusto e eficiente?
