**### Pacote de Ativação de Missão (PAM) V1.0**

* **Sumário da Missão (O Quê):** Realizar uma auditoria forense completa do endpoint `GET /api/cobrancas` para identificar a causa raiz exata pela qual o `valorTotal` de uma proposta está a ser exibido incorretamente como R$ 0.
* **Intenção Estratégica (O Porquê):** O Arquiteto identificou uma falha de integridade de dados financeiros crítica. Antes de qualquer correção, precisamos de um diagnóstico 100% preciso de como este valor está a ser calculado e onde a lógica está a falhar.
* **Histórico Relevante (Consulta Obrigatória):** Nossas auditorias anteriores focaram na lista de propostas, mas não na precisão dos cálculos financeiros dentro dela. A sua análise deve focar exclusivamente na lógica de agregação de valores.
* **Modelo Mental (Como se Encaixa):** Você é um Auditor Financeiro de Sistemas. Sua tarefa é analisar o "motor de cálculo" no backend e provar, com evidências de código, por que um valor de R$ 20.000 se transforma em R$ 0.

**IMPLEMENTAR:**
Um relatório de auditoria detalhado. **NÃO FAÇA MODIFICAÇÕES.**

**CURRENT STATE:**
Uma proposta com um valor financiado de R$ 20.000 está a ser exibida na "Tela de Cobranças" com um `valorTotal` de R$ 0.

**EXPECTED (Estado Final de Sucesso):**
Um relatório de auditoria que aponte a linha de código exata e a falha lógica que está a causar o cálculo incorreto do `valorTotal`.

**CONSTRAINTS (Roadmap de Auditoria Forense):**

**1. Auditoria da Lógica de Cálculo no Backend (`GET /api/cobrancas`):**
    -   **Ação:** Localize o endpoint e a lógica que calcula ou busca os dados para a "Tela de Cobranças".
    -   **Relatório 1 (A Evidência Crítica):** Encontre e apresente o trecho de código exato que define o campo `valorTotal` no objeto de resposta para cada proposta.
    -   **Análise de Causa Raiz:** Com base no código, formule uma hipótese para a falha. A lógica está a ler da coluna errada? Está a falhar em converter um tipo de dado (ex: `string` para `number`)? Existe uma condição `if` que está a zerar o valor indevidamente?

**2. Verificação Cruzada com o Banco de Dados:**
    -   **Ação:** Para a proposta de teste (a de R$ 20.000), execute uma query `SELECT` direta no banco de dados para verificar os valores nas colunas relevantes (ex: `valor_total_financiado`, `valor`, etc.).
    -   **Relatório 2:** Apresente o resultado desta query para confirmar que o valor de R$ 20.000 existe e está correto no banco de dados.

**Relatório Final:**
O critério de sucesso é um relatório forense que nos dê um diagnóstico definitivo, com evidências de código e do banco de dados, explicando por que o `valorTotal` está a ser exibido como zero.
