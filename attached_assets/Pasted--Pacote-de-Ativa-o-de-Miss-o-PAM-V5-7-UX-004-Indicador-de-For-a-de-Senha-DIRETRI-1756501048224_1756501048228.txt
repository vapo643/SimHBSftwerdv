**IMPLEMENTAR:**
User Story UX-004: Implementar Feedback Visual de Força de Senha

**CONTEXT:**
Nossa auditoria de UX revelou que o formulário de criação de conta não fornece nenhum feedback ao usuário sobre a qualidade da senha que ele está digitando. O usuário não sabe se sua senha é forte o suficiente ou se cumpre os requisitos mínimos do sistema, o que pode levar à criação de senhas fracas ou a erros de validação após a submissão.

**CURRENT STATE:**
O formulário em `client/src/pages/login.tsx` possui um campo de senha, mas não há componentes ou texto adicionais para indicar os critérios da senha ou sua força em tempo real.

**EXPECTED (Estado Final de Sucesso):**
1.  **Exibição dos Critérios:** Abaixo do campo de senha, um texto informativo deve ser exibido, listando os critérios mínimos para uma senha ser válida (ex: "Mínimo de 6 caracteres").
2.  **Indicador Visual de Força:** Um componente de barra de progresso (ou similar) deve ser exibido, mudando de cor e preenchimento para refletir a força da senha digitada em tempo real (ex: Vermelho para "Fraca", Amarelo para "Média", Verde para "Forte").
3.  **Lógica de Validação:** Uma biblioteca de validação de força de senha (como `zxcvbn`) ou uma lógica customizada deve ser integrada para alimentar o indicador visual.
4.  **Prova de Sucesso:** Você deve fornecer um screenshot do formulário de login mostrando o campo de senha, os critérios visíveis e o indicador de força de senha em ação (ex: no estado "Forte").

**CONSTRAINTS (Roadmap de Implementação):**
1.  **Localizar Componente Alvo:** O foco principal é o arquivo `client/src/pages/login.tsx`.
2.  **Integrar Biblioteca de Validação:** Se for usar uma biblioteca externa como `zxcvbn`, adicione-a como dependência.
3.  **Implementar Lógica de Feedback:**
    * Crie um novo estado no componente (`useState`) para armazenar a força da senha (ex: `passwordStrength`).
    * No `onChange` do campo de senha, chame a função de validação e atualize o estado `passwordStrength`.
4.  **Refatorar JSX:**
    * Adicione o texto com os critérios mínimos abaixo do campo de senha.
    * Adicione um componente de `Progress` do `shadcn/ui` ou um componente customizado.
    * Vincule a aparência (cor, valor) deste componente ao estado `passwordStrength`.
5.  **Validar e Apresentar Evidência:** Teste o formulário digitando senhas de diferentes complexidades para garantir que o indicador reage corretamente. Capture e apresente o screenshot do resultado final.
6.  **Verificação Final:** Execute o protocolo `7-CHECK FULL`.
