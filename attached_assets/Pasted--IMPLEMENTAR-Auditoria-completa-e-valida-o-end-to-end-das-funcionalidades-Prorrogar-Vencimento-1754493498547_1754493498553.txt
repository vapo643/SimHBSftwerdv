**IMPLEMENTAR:** Auditoria completa e validação end-to-end das funcionalidades "Prorrogar Vencimento" e "Desconto para Quitação" na "Tela de Cobranças".

**CONTEXT:** A lógica complexa para estas ações foi implementada. Agora, precisamos de uma verificação rigorosa para garantir que cada passo do fluxo, especialmente as chamadas à API do Banco Inter e as atualizações no nosso banco de dados, está funcionando exatamente como esperado.

**CURRENT STATE:** Os botões e modais para as ações estão funcionais na interface.

**EXPECTED (Estado Final de Sucesso):** Um relatório de auditoria detalhado que confirme, com evidências claras (logs do console), que ambas as funcionalidades estão operando corretamente de ponta a ponta.

**TESTE (Roadmap de Auditoria Obrigatório):**

**Parte 1: Auditoria da Ação "Prorrogar Vencimento"**
1.  **Instrumentação:** Antes de executar o teste, adicione logs `console.log` em pontos-chave:
    - No **endpoint do backend** que lida com a prorrogação, logue o `codigoSolicitacao` e a `novaDataVencimento` recebidos do frontend.
    - Dentro da função `editarCobranca` no `interBankService`, logue o **payload exato** que está sendo enviado para a API do Inter.
    - Logue a **resposta completa** (status e corpo) recebida da API do Inter.
2.  **Execução:**
    - Use um usuário `ADMIN` e selecione um boleto ativo de uma proposta de teste.
    - Utilize a funcionalidade para prorrogar o vencimento para uma nova data.
3.  **Validação End-to-End:**
    - Após a ação, o seu script de auditoria deve **automaticamente** chamar a função `recuperarCobranca` do `interBankService` para o mesmo boleto.
    - **Relate:** Compare a `dataVencimento` retornada nesta chamada de verificação com a nova data que você enviou. Elas devem ser idênticas.
4.  **Validação do Banco de Dados Local:**
    - **Relate:** Confirme que a `dataVencimento` na nossa tabela `inter_collections` também foi atualizada para a nova data.

**Parte 2: Auditoria da Ação "Desconto para Quitação"**
1.  **Instrumentação:** Adicione logs `console.log` para os seguintes eventos:
    - O **"Valor Restante da Dívida"** calculado no início do fluxo.
    - O `codigoSolicitacao` de **cada boleto antigo** que está sendo cancelado.
    - O **payload completo** de **cada novo boleto** que está sendo criado.
    - A **resposta da API do Inter** para cada operação de cancelamento e criação.
2.  **Execução:**
    - Use um usuário `ADMIN` e uma proposta de teste com múltiplos boletos ativos.
    - Execute o fluxo completo de quitação, aplicando um desconto e gerando 2 novas parcelas.
3.  **Validação End-to-End:**
    - Após a criação dos novos boletos, seu script deve usar a função `recuperarCobranca` para **consultar cada um dos novos boletos** na API do Inter.
    - **Relate:** Confirme que os novos boletos existem no sistema do Inter e que seus valores e datas estão corretos.
4.  **Validação do Banco de Dados Local:**
    - **Relate:** Confirme que os boletos antigos na tabela `inter_collections` foram corretamente marcados como inativos (`isActive: false`).
    - **Relate:** Confirme que os novos boletos foram inseridos na tabela `inter_collections`.
    - **Relate:** Confirme que o registro de auditoria foi criado com sucesso na tabela `historico_observacoes_cobranca`.

**Relatório Final:** Apresente um sumário claro dos resultados da auditoria para cada parte (Prorrogação e Quitação), confirmando se cada passo da validação foi bem-sucedido.
