* **Sumário da Missão (O Quê):** Realizar uma auditoria de integridade de dados de ponta-a-ponta nos fluxos de CRUD (Create, Read, Update, Delete) para as entidades de negócio principais: Parceiros, Lojas, Produtos e Tabelas Comerciais.
* **Intenção Estratégica (O Porquê):** Proativamente caçar e eliminar falhas silenciosas de persistência de dados. * **Histórico Relevante (Consulta Obrigatória):** A missão `FIX-CRUD-001` corrigiu uma falha crítica de roteamento. Esta missão irá verificar se outras entidades foram afetadas por problemas similares ou outros tipos de bugs de persistência.

**AUDITAR E VALIDAR:**
Integridade dos Fluxos de CRUD para Entidades de Negócio

**CONTEXT:**
Após corrigir uma falha grave na criação de usuários, suspeitamos que outros formulários de criação e edição na aplicação possam sofrer de problemas similares, onde a UI parece funcionar, mas os dados não são salvos corretamente no banco. Esta missão é uma varredura completa para garantir que todas as nossas operações de escrita de dados são robustas e confiáveis.

**CURRENT STATE:**
A aplicação possui formulários e APIs para gerenciar Parceiros, Lojas, Produtos e Tabelas Comerciais. A integridade de ponta-a-ponta desses fluxos não foi auditada após a descoberta da falha no fluxo de usuários.

**EXPECTED (Estado Final de Sucesso):**
Você deve produzir um **Relatório de Auditoria de CRUD** consolidado. Para **cada uma** das quatro entidades (Parceiros, Lojas, Produtos, Tabelas Comerciais), o relatório deve conter:
1.  **Mapeamento do Fluxo de Dados (Criação):**
    * O caminho do arquivo do componente do formulário de criação (ex: `PartnerForm.tsx`).
    * O endpoint da API `POST` que ele chama (ex: `/api/admin/partners`).
    * O nome do serviço e método do backend que lida com a criação.
2.  **Validação da Criação:**
    * Execute uma simulação de criação de uma nova entidade.
    * Forneça uma query `SELECT` e seu resultado, provando que o novo registro foi persistido corretamente no banco de dados.
3.  **Mapeamento do Fluxo de Dados (Atualização):**
    * O endpoint da API `PUT` ou `PATCH` para atualização.
    * O nome do serviço e método do backend que lida com a atualização.
4.  **Validação da Atualização:**
    * Execute uma simulação de atualização da entidade recém-criada.
    * Forneça uma query `SELECT` e seu resultado, provando que os campos foram atualizados corretamente no banco.
5.  **Veredito de Conformidade por Entidade:** Uma declaração clara para cada entidade: "✅ CONFORME" ou "❌ FALHA", com detalhes caso uma falha seja encontrada.

**CONSTRAINTS (Roadmap de Implementação):**
1.  **Ciclo de Auditoria - Parceiros:**
    * Analise o fluxo de criação/edição de Parceiros, seguindo os 5 pontos da seção "EXPECTED".
2.  **Ciclo de Auditoria - Lojas:**
    * Repita o processo de auditoria completo para a entidade Lojas.
3.  **Ciclo de Auditoria - Produtos:**
    * Repita o processo de auditoria completo para a entidade Produtos.
4.  **Ciclo de Auditoria - Tabelas Comerciais:**
    * Repita o processo de auditoria completo para a entidade Tabelas Comerciais.
5.  **Compilar Relatório Final:** Organize todas as suas descobertas em um único relatório estruturado.
