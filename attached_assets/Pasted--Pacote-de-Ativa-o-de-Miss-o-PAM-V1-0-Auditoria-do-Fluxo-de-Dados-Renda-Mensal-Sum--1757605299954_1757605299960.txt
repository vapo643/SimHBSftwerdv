### Pacote de Ativação de Missão (PAM) V1.0 - Auditoria do Fluxo de Dados 'Renda Mensal'

* **Sumário da Missão:** Realizar uma auditoria forense de ponta a ponta para mapear o fluxo de dados do campo `clienteRenda`. O objetivo é identificar o ponto exato no código (frontend ou backend) onde o valor numérico está a ser transformado ou dividido incorretamente.

* **Hipótese / Ponto de Falha:** Existe uma lógica de formatação monetária ou conversão para centavos a ser aplicada indevidamente ao campo `clienteRenda`. Suspeita-se de uma operação de divisão (`/ 100` ou `/ 10000`) na camada de serviço do backend ao salvar os dados, ou na camada de exibição ao formatar os dados para o analista.

---

**Evidência de Conformidade Requerida (A "Prova Irrefutável"):**

**NÃO MODIFIQUE NENHUM CÓDIGO.** A sua única missão é LER, ANALISAR e APRESENTAR.

**1. Análise do Frontend (Submissão):**
    a. Apresente o código do ficheiro `client/src/pages/nova-proposta.tsx`.
    b. Identifique e apresente a definição do schema `zod` para o campo `clienteRenda`.
    c. No método de submissão do formulário, mostre a linha exata onde o valor de `clienteRenda` é processado antes de ser enviado para a API (`POST /api/propostas`).

**2. Análise do Backend (Persistência):**
    a. Apresente o código do controlador que recebe a requisição `POST /api/propostas` (provavelmente em `server/modules/proposal/presentation/proposalController.ts`).
    b. Mostre como o valor de `clienteRenda` é extraído do `req.body` e passado para a camada de serviço ou repositório para ser salvo na base de dados. Existe alguma transformação matemática aqui?

**3. Análise do Backend (Leitura para o Analista):**
    a. Identifique e apresente o código do controlador e do serviço que buscam os dados de uma proposta para a tela do analista (provavelmente um método `getById` ou similar).
    b. Rastreie o campo `clienteRenda` desde a sua leitura da base de dados até ao momento em que é retornado na resposta da API. Existe alguma transformação matemática ou de formatação a ser aplicada neste fluxo?

**4. Conclusão da Auditoria:**
    a. Com base na sua análise de ponta a ponta, aponte o ficheiro e a linha de código exatos onde a transformação de `300000` para `30` está a ocorrer e explique a lógica defeituosa.
