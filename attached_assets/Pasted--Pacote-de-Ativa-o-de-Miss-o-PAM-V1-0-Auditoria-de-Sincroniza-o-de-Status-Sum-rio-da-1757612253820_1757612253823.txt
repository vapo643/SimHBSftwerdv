### Pacote de Ativação de Missão (PAM) V1.0 - Auditoria de Sincronização de Status

* **Sumário da Missão:** Realizar uma auditoria forense de ponta a ponta para diagnosticar por que a "Fila de Formalização" e a tabela "Gerenciar Status" estão a exibir status de proposta obsoletos, em vez dos novos status canónicos do sistema.

* **Hipótese / Ponto de Falha:** A hipótese principal é que os componentes de frontend responsáveis por estas visualizações estão a utilizar um `enum`, uma lista estática ou uma função de mapeamento de "tradução" que contém os nomes dos status antigos, em vez de renderizar diretamente os valores de status reais que são retornados pela API do backend.

* **Fonte da Verdade (Status Canónicos):** `CCB_GERADA`, `AGUARDANDO_ASSINATURA`, `ASSINATURA_PENDENTE`, `ASSINATURA_CONCLUIDA`, `BOLETOS_EMITIDOS`.

---

**Evidência de Conformidade Requerida (A "Prova Irrefutável"):**

**NÃO MODIFIQUE NENHUM CÓDIGO.** A sua única missão é LER, ANALISAR e APRESENTAR.

**1. Análise do Backend (Verificação da Fonte dos Dados):**
    a. Identifique o endpoint da API que alimenta a "Fila de Formalização".
    b. Apresente a consulta da base de dados executada por este endpoint. Confirme que ela seleciona o campo `status` da tabela `propostas`.
    c. Apresente um exemplo do objeto de proposta que é retornado na resposta da API. Confirme que o campo `status` contém os novos valores canónicos (ex: `"CCB_GERADA"`).

**2. Análise do Frontend (Verificação da Camada de Apresentação):**
    a. Identifique e apresente o código-fonte completo dos componentes React que renderizam a "Fila de Formalização" e a tabela "Gerenciar Status".
    b. Dentro destes componentes, localize a secção de código exata que renderiza a coluna ou o campo "Status".
    c. **Análise Crítica:** Existe um `enum` de `ProposalStatus`, um `switch-case`, um objeto de mapeamento (ex: `statusMap`), ou uma função de tradução (ex: `translateStatus(status)`) a ser utilizada para converter o valor do status vindo da API para um texto a ser exibido? Se sim, apresente o código completo desta lógica de mapeamento/tradução.

**3. Conclusão da Auditoria:**
    a. Com base na sua análise, aponte o ficheiro e as linhas de código exatas no **frontend** que contêm a lista de status obsoleta e que precisam de ser atualizadas.
